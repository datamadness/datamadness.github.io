Creating custom transformer for your scikit-learn pipeline

Having a pipeline to transform your data is an absolute must for any machine learning project. Besides speeding up the intake of new data and providing much cleaner code, well designed pipeline allows us to include the initial data transformation steps in hyperparameter tuning. While packages like scikit-learn include many data transformers out of the box, you will always need to include some custom transformations. In this case, it is highly recommended to build a custom transformer based on the underlying framework so it can be included in the final pipeline.

This example will demonstrate how to build a custom transformer for scikit-learn. Specifically, the transformer will take Pandas DataFrame and drop any columns that have more than x% of nan values. Afterwards, it drops any rows with nan values to clean up the full DataFrame.

Framing this operation as a scikit-transformer will allow us to include it in the pipeline as opposed to running it as separate transformation step. Moreover, we will be able to use the x% cutoff limit as a tunable hyperparameter when using native scikit classes such as GridSearchCV.

Building the Transformer

The transformer class requires us to create three methods consistent with any other scikit transformer:

__init__ This method initiates the transformer and allows is to introduce hyperparameters. In this case we have only a single hyperparameter defining the maximum acceptable fraction of rows with nan values. Anything above and the full column will be removed.

    def __init__(self, max_bad_rows_pct = 5):
        self.max_bad_rows_pct=max_bad_rows_pct

fit The fit methods allows us to train the transformer on specific dataset without transforming it. For example, we could train it on “Train” data and then apply it on both “Train” and “Test” datasets to assure consistency.

    def fit(self, X, y = None):
        #Find which columns have more than acceptable fraction of nans
        cutoff = int(len(X) * self.max_bad_rows_pct / 100)
        badColumns = X.isnull().sum(axis = 0)
        badColumns = badColumns.index[badColumns > cutoff].tolist()
        print(badColumns)
        self.badColumns = badColumns

transform

    def transform(self, X, y=None):
        #Drop columns with too many nans
        X.drop(columns = self.badColumns, inplace = True)
        #Drop any remaining rows with nan values
        X.dropna(inplace = True)
        return X

That is it! Simple, yet powerful way to introduce custom transformers to your pipeline. Here is the full class when we put it together:

from sklearn.base import BaseEstimator, TransformerMixin

class NanRemover(BaseEstimator,TransformerMixin):
    def __init__(self, max_bad_rows_pct = 5):
        self.max_bad_rows_pct=max_bad_rows_pct
        
    def fit(self, X, y = None):
        #Find which columns have more than acceptable fraction of nans
        cutoff = int(len(X) * self.max_bad_rows_pct / 100)
        badColumns = X.isnull().sum(axis = 0)
        badColumns = badColumns.index[badColumns > cutoff].tolist()
        print(badColumns)
        self.badColumns = badColumns
        
    def transform(self, X, y=None):
        #Drop columns with too many nans
        X.drop(columns = self.badColumns, inplace = True)
        #Drop any remaining rows with nan values
        X.dropna(inplace = True)
        return X

Use case example:

myTransformer = NanRemover(max_bad_rows_pct = 10)
trainDF = myTransformer.fit_tranform(trainDF)
testDF = myTransformer.transform(testDF)
